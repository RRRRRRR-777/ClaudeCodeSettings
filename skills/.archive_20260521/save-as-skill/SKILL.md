# Save As Skill

## 目的

対話中に再利用できそうなパターンやワークフローをSkillとして保存し、再現性と一貫性を向上させる。

---

## 実行手順

このSkillが呼び出されたとき、以下の手順を実行する。

### 1. 対話内容の分析

直近の対話から以下を検出：
- 繰り返し現れるパターン（2回以上）または繰り返されると想定される内容
- 成功したワークフロー
- 標準化できそうな手順

### 2. パターンの分類

検出されたパターンを以下に分類：

| カテゴリ | 説明 |
|----------|------|
| 実装パターン | コード生成・修正の標準手順 |
| デバッグパターン | エラー修正の定型的な手順 |
| レビューパターン | コードレビューのチェックリスト |
| 設定パターン | 環境構築・設定手順 |
| テストパターン | テスト作成・実行のワークフロー |

### 3. Skill候補の提示

検出されたパターンを以下の形式で提示：

```
## 検出された再利用可能なパターン

### 1. [コマンド名] ([カテゴリ])
**検出理由**: [理由]
**対象**: [対象]
**成功率**: [成功率]

### 2. ...
```

### 4. ユーザー確認

以下の選択肢で確認：
1. すべて作成
2. 選択して作成
3. 内容を修正して作成
4. キャンセル

### 5. Skillファイルの生成

`.claude/skills/[name]/SKILL.md` を以下の形式で作成：

```markdown
---
name: [skill-name]
description: [簡潔な説明]
---

# [タイトル]

## 概要

[このSkillの目的と効果]

## 実行手順

[具体的なステップ]

## 使用例

```
User: /[skill-name] [リクエスト]

Agent: [応答]
```

## 注意事項

- [注意点1]
- [注意点2]
```

---

## 保存場所

| 種類 | 場所 |
|------|------|
| グローバル | `~/.claude/skills/` |
| ローカル | `{プロジェクト}/.claude/skills/` |

ユーザーに確認して作成する

---

## 制約事項

- 機密情報（APIキー、パスワード等）は含めない
- プロジェクト固有の情報は汎用化する
- 不明な点は `/ask` を使用して確認する

---

## このSkillの一文定義

対話パターンを検出し、再利用可能なSkillとして保存する
