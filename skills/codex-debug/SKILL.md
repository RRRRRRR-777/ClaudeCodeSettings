# Codex Debug Skill

OpenAI Codex CLI を Agent tool（バックグラウンド）で実行し、エラーの根本原因究明・デバッグを行う。`codex exec --full-auto` による自由形式の調査で、Codex 処理中もユーザーと会話を継続できる。

## 手順

STEP 1 → STEP 3 を上から順に実行する。

### STEP 1: 症状を集める
次を埋めて、STEP 2 の依頼文を組み立てる材料にする。

| 項目 | 内容 |
|------|------|
| エラー内容 | メッセージ・スタックトレース |
| 期待 / 実際 | 期待動作と実際の動作 |
| 再現・範囲 | 再現手順・対象ファイル |

依頼文の型（症状に応じて1つ選ぶ）:
- **ランタイムエラー**: `以下のエラーをデバッグ: {エラーメッセージ}。スタックトレースを解析し、根本原因を特定`
- **ロジックバグ**: `期待動作: {期待}、実際の動作: {実際}。ロジックの問題を調査`
- **パフォーマンス問題**: `処理が遅い箇所を特定。ボトルネック分析と改善案を提示`

### STEP 2: 単一 Agent で発行する
`run_in_background=true` で1つの Agent を発行する。`{依頼文}` に STEP 1 で組み立てた文を入れる。

```
Agent(
  name="codex-debugger",
  description="Codexデバッグ実行",
  run_in_background=true,
  prompt="Bashツールで以下のコマンドを実行してください（timeout: 600000）。結果のstdoutをそのまま返してください。

codex exec --full-auto '{依頼文}'
"
)
```

### STEP 3: 完了後に報告する
完了通知を待ち（その間ユーザーと会話継続可）、stdout を下の DebugResult に流し込んで報告する。

## DebugResult（STEP 3 で使う素材）
| フィールド | 説明 |
|-----------|------|
| 症状 | 報告されたエラー・問題の要約 |
| 原因 | 特定された根本原因 |
| 調査過程 | 調査手順と確認した内容 |
| 解決策 | 修正方法・コード変更案 |
| 予防策 | 再発防止のための提案 |
